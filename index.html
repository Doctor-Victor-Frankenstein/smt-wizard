The control account has administrative authority over the created token. All subsequent changes to the token will require the authorities of the control account. For more information on control account best practices review the applicable sections of the SMT Whitepaper.

The launch time determines when tokens become active. If the token features an ICO (Initial Coin Offering), the launch time must be on or after the contribution end time. Once a token launches it becomes transactable on the Steem blockchain similar to STEEM and STEEM Power.

Tokens

The max supply is the total number of tokens that can exist on the blockchain. The supply is created through the ICO process or through inflation with token emissions. By adding a maximum supply, you ensure that you can never inflate more than the given amount of tokens.

The precision will dictate how many decimal places a particular token will have available to it. For example, the STEEM coin has a precision of 3. This allows users to send and receive denominations as low as 0.001 STEEM.

Initial Coin Offering

The contribution begin time specifies when a particular ICO begins. When the contribution begin time is reached, the token opens up the ICO process to contributors. At this time users may begin to make contributions.

The contribution end time determines when the ICO process is complete. When the contribution end time is reached users can no longer make contributions and the ICO is evaluated.

STEEM

The minimum contribution is the number of STEEM required for the ICO to be considered successful. If the minimum contribution is not met, the token is moved in to a failure status and the contributors are refunded. If the contributions exceed the minimum contribution, the ICO is considered successful and moves on to the launch phase.

The minimum unit ratio is the number of tokens created per STEEM unit in a R-for-1 ratio. The unit ratio is dynamic as per the token destination unit map, but this ratio has a floor specified by the minimum unit ratio.

The maximum unit ratio is the number of tokens created per STEEM unit in a R-for-1 ratio. The unit ratio is dynamic as per the token destination unit map, but this ratio has a ceiling specified by the maximum unit ratio.

Voting Parameters

The allow voting parameter toggles the Proof-of-Brain voting feature for the given token. When disabled, users will not be able to receive inflation through the Proof-of-Brain voting mechanism.

The allow downvoting parameter toggles the Proof-of-Brain downvoting feature for the given token. When disabled, users will not be able to downvote posts.

Seconds

The cashout window changes how long it takes for a user to receive their payout. Steem has a one week cashout window, after a user has voted and a week has passed the user is then able to claim their rewards.

Seconds

The reverse auction window changes the window of time after a post is created where is user is penalized for voting too early. This is usually used as a counter measure to automated voting bots and other such similar services.

Seconds

The vote regeneration period sets the amount of time used in the calculation of the user vote regeneration. When combined with the votes per period, this can be used to determine how quickly a users votes regenerate or how often users can cast a vote.

Votes

The votes per regeneration period sets the amount of votes a user will regenerate within the regeneration period. When combined with the votes per regeneration period, this can be used to determine how quickly a users votes regenerate or how often users can cast a vote.

The content constant is essentially an arbitrary number which is used during a reward curve calculation. Depending on the chosen reward curve this number can have a variety of effects.

%

The percent curation rewards determines how much inflation is used to reward curators rather than content creators. If you have a 25% percent curation rewards then the content creator reward will be 75%.

The author reward curve determines which curve will be used to calculate author rewards. The author reward curve used in combination with the content constant can be used to determine the amount of rewards a voter is eligible for.

The curation reward curve determines which curve will be used to calculate curation rewards. The curation reward curve used in combination with the content constant can be used to determine the amount of rewards a voter is eligible for.

Token Emissions

Tier

STEEM

The STEEM cap is the STEEM value ceiling that this particular ICO tier is applicable. When the cap is exceeded, the next ICO tier goes in to effect. The last ICO tiers Steem unit cap is effectively a cap on the entire ICO process. No more contributions will be accepted when it is reached.

The Steem unit map defines the destination of STEEM units during the ICO phase. For example, if the total number of STEEM units is 3 split evenly among 3 founders, then each of the founders would receive ⅓ of the total STEEM contributors during this particular tier of the ICO.

The Token unit map defines the destination of token units during the ICO phase. For each STEEM unit during the ICO contribution phase, the total number of token units will be created and dispersed among each destination.

Token Emission Schedule

The schedule time determines the beginning of the token emission schedule. This is the date and time of the first token emission within the token emission schedule.

The left endpoint time specifies the left most data point of the token emission schedule slope. The left endpoint must be on or after the schedule time and on or before the right endpoint.

The right endpoint time specifies the right end of the token emission schedule slope. The right endpoint must be on or after the left endpoint and on or before schedule end time.

Seconds

The interval seconds specifies the number of seconds between token emissions. The interval seconds must be greater than 21600 (6 Hours).

The emission count specifies the number of emissions within a token emission schedule. The first emission is on the schedule time, each subsequent emission is the last emission plus the interval seconds.

Bits

The relative denominator is the number of left bit shifts to use as the denominator for relative inflation. We use a bitshift operation to avoid any floating point operation or precision loss on the blockchain.

The left endpoint amount defines the absolute value of inflation on the left endpoint. Depending on the floor emissions setting, if an emission lands precisely on the left endpoint this is the amount of tokens created in the emission.

The right endpoint amount defines the absolute value of inflation on the right endpoint. Depending on the floor emissions setting, if an emission lands precisely on the right endpoint this is the amount of tokens created in the emission.

The left endpoint numerator defines the relative value of inflation on the left endpoint. Used in conjunction with the relative denominator bits and the current token supply, this determines the inflation at the left endpoint.

The right endpoint numerator defines the relative value of inflation on the right endpoint. Used in conjunction with the relative denominator bits and the current token supply, this determines the inflation at the right endpoint.

The floor emissions parameter will determine whether the absolute inflation or relative inflation values will be used. If this parameter is enabled then the blockchain will use whatever value is lower.

The emission unit map determines where newly minted tokens will be sent to. This can be a specific users account as well as a handful of special destinations that are available in the drop down menu.
